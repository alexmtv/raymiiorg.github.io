Encryption on EVERY cloud service!

  • Home /
  • Articles /
  • Encryption on EVERY cloud service!
Author: Remy Van Elst
Date: 01-06-2012

As you maybe know, there are a lot of cloud providers out there. They all offer kind of the same thing, a place for you to store your files. Main differences are the storage space you get and collaborative/syncing features. And, a lot of providers say they are secure, but they do not have "Trust No One" encryption (which means, you have the key to your data and only you can access it. If you loose the key or password, you're out of luck.) Those companies, like dropbox or google drive at the time of writing, all state that in their terms of service, and/or you are not able to check the source code to verify they can or cannot access your data. What we are going to do to prevent this? Very simple, provide our own pre-internet encryption with truecrypt.

We are going to make an encrypted volume of which only we know the password. Then we are going to sync it to the cloud service, and when we need it we can download it everywhere and open it with truecrypt and our password.

Supported services

This will work on any cloud service, even an FTP server, or usenet or wherever you can upload files. The only two cloud services I recommend (and use) are Spideroak and Wuala.

Install

First download and install truecrypt:

download the setup from their website for your platform or install it via your package manager (eg: yaourt -S truecrypt, emerge truecrypt, yum install truecrypt, apt-get install truecrypt, equo install truecrypt).
I will not explain here how to install it, on windows/mac it is nothing more than clicking next-next-next-done and linux is just as easy.

Startup

Start up truecrypt and click on "Create Volume":

Volume Creation

We need to create an encrypted file where we are going to store our data in. The volume creation has to be done only once. Select: "Create an encrypted file container":

Select: "Standard TrueCrypt Volume":

Click on: "Select File" and choose a location where to create the file. This is what you would place in your cloud service folder (the "Dropbox" folder, or a folder that is synced with the cloud service):

You can name the file anything you want. I have used the filename "finance.tc" in this example (For bank and finance files).
Click "Next":

Select an encyrption algorithm. You can use AES for almost everything. If you choose a double or triple algorithm truecrypt can work slower. In this example I choose "AES-Twofish":

Click "Next" Select a volume size.
It is important to think about what data you are going to store. Music you've ripped or downloaded does not need to be encrypted. Music you created might be. Word documents with sensitive information should be, scans of your passport and driving license should also be.
There is a difference in size for those files. For those word documents, you could do with a 50 mb file. If you use a password manager like Keepass, you could fit it in a 5 MB file.
If you are going to encrypt all your "private" pictures (HQ), you might need a 40GB file (raw files, very high resolution). Remember that you are syncing this to a cloud service which might have limited space. If you have a 5 GB google drive, then do not create a 10GB volume.
Do note that the size of the file cannot be changed later on. You then need to create a new file.

In this example I've chosen a 20 MB file for my finance files:

Click "Next"

Now we are going to enter a password. It is very important that you choose a strong password which you can remember, but which other people (for whom you are hiding the stuff you are encrypting) cannot guess. SO NOT THE NAME OF YOUR DOG OR YOUR DAUGHTERS BIRTHDAY!!!/rant, Ive seen a lot of people which such passwords in my IT support days... Steve Gibson from GRC.com has a very nice password generator. Click here to use it..
If you've filled in a password click Next:

Select "FAT" as the filesystem type. If you are going to use it on Linux exclusivly you might set EXT3 or EXT4, but if you are going to use it on Windows, Mac or on mixed platforms, use FAT:

Click "Next" Now move your mouse in that window for a while and then click "Format":

When it is done, click OK:

We are done for now, so click "Exit"

Mounting the volume

We are now going to mount the volume. That means that our operating system will be able to see the encrypted volume and add data to it. In the truecrypt window click on a free device slot. This might look a bit different on windows, there are drive letters there instead of numbers as in the screenshot (which is gentoo linux).

Then click on Select file and select your created truecrypt volume:

Now enter your password:

It might ask you for your root/admin password, this is to mount the volume. If you get a permission needed (UAC prompt) then allow it:

I had to do the mount procedure twice because it failed the first time with a file descriptor error. I think that is my linux system, but it might happen to you. Just try it again then. Now the volume is mounted:

Doing Data

On windows, you will have a new drive (F:) where you can add data. On Linux you have a new folder which is shown in the truecrypt window as "Mount Directory". Go to that folder and add or edit or delete some files you want to sync:

Unmount the volume

Now we have added or removed some data and to be sure the truecrypt volume is synced correct we need to dismount/unmount the file. In the main truecrypt window select your mounted volume and click "Dismount":

Now it will be synced with your cloud service, fully encrypted!

What now

Every time you want to do something with the data you've encrypted, you need to do the steps "Mounting the volume", "Doing Data" and "Unmounting the Volume".
If you are on a new computer you need to install truecrypt first. If you cannot install truecrypt, follow the steps here for a portable version of truecrypt.

Done

We are done now. You have an encrypted file, on a "insecure" cloud service. Everybody's happy now!

Thanks to Steve Gibson for the idea, and his excellent podcast Security Now.
