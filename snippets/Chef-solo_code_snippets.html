Chef-Solo code snippets

  • Home /

Author: Remy Van Elst
Date: 21-09-2012

This article is a collection of snippets for Chef. Chef is an open-source systems integration framework supporting Linux, Windows, BSD and more. For more info about chef, click here

These snippets are tested and working with Ruby 1.9.2 and Chef-solo on Ubuntu 10.04, 12.04 and CentOS 5 and 6.

Do something on a server, based on server name

Create a new recipe, name it server-specific-config for example. In the data_bags/servers/$servername.json file, make sure the name matches this one.

server-specific-config/recipes/default.rb
attribs = get_configuration_attribs(node.hostname)
server_role = attribs['node_data']['role'] 
server_name = attribs['id'] 

case server_name
  when "vps5", "vps6", "vps7"
    # these are the SQL servers which should have specific databases as defind in mydatabases recipe
    include_recipe "mysql::server"
    include_recipe "database"
    include_recipe "mydatabases"
  when "vps4", "vps3", "vps2"
    #these are the webservers which act as reverse-proxy, and should not have lighttpd
    include_recipe "nginx"
    include_recipe "raymiiorg-revproxy"
end
data_bags/servers/$servername.json
{
    "id": "vps5",
    "node_data": {
        "description": "MySQL database slave 1",
        "role": "database"
        }
}
Do something on a server, based on server role

Create a new recipe, name it server-specific-config for example. In the data_bags/servers/$servername.json file, make sure the role matches this one. See below for an example $servername.json file.

server-specific-config/recipes/default.rb
attribs = get_configuration_attribs(node.hostname)
server_role = attribs['node_data']['role'] 
server_name = attribs['id'] 

case server_role
  when "webserver"
    #non-reverse proxy servers should have lighttpd.
    include_recipe "lighttpd"
    include_recipe "mysql::client"
  when "database"
      ##generic database servers should be empty
      include_recipe "mysql::server"
      include_recipe "database"
      gem_package "mysql" do
        action :install
        #ignore_failure true
      end
      package "memcached" do
        action :install
      end
end
data_bags/servers/$servername.json
{
    "id": "vps9",
    "node_data": {
        "description": "Lighttpd webserver 2",
        "role": "webserver"
        }
}
MySQL - Create multiple databases and create and add a user to that database

Create a new recipe, name it mydatabases for example. Make sure you include this recipe on a database server (see above). This requires the mysql and the database recipe.

mydatabases/recipes/default.rb
include_recipe "mysql::server"
include_recipe "database"

mysql_connection_info = {:host => "localhost", :username => 'root', :password => node['mysql']['server_root_password']}

#DB NAME, USER AND PASSWORD ARE REQUIRED!!!

node[:default][:databases].each do |db|

    mysql_database db[:name] do
        connection mysql_connection_info
        action :create
    end

    mysql_database_user db[:user] do
        connection mysql_connection_info
        password db[:password]
        action :create
    end

    mysql_database_user db[:user] do
        connection mysql_connection_info
        password db[:password] 
        database_name db[:name]
        privileges [:all]
        host '%'
        action :grant
    end
end
mydatabases/attributes/default.rb
default[:default][:databases] = [

Install multiple packages on a server

This can be in a different recipe, or in the server-specific-config recipe, for example just for server vps5.

server-specific-config/attributes/default.rb
default[:vps5][:packages_to_install] = [
    "htop",
    "fail2ban",
    "vim",
    "nano",
    "wget",
    "curl",
    "git-core",
    "build-essential",
    "mailutils",
    ]
server-specific-config/recipes/default.rb
default[:vps5][:packages_to_install].each do |p|
  package p do
    action :install
  end
end
Remove a list of users

Lets say we do not want all those users defaulting on an ubuntu server system. Lets remove them via the server-specific-config recipe.

server-specific-config/recipes/default.rb
default[:vps5][:users_to_delete].each do |user|
        user "#{user}" do
            action :remove
            username "#{user}"
        end
end
server-specific-config/attributes/default.rb
default[:vps5][:users_to_delete] = [
    "games",
    "lp",
    "mail",
    "news",
    "uucp",
    "proxy",
    "list",
    "irc",
    "gnats",
    "landscape",
    ]

Execute a command as another user than the one doing the chef-solo run
execute "Send email on completion" do
    command "mail -s \"Chef run on node #{node} complete\" email@mail.com < /var/chef/chef.log"
    user "remy"
    action :run
end

Add remote logging to rsyslog
ruby_block "Adding remote loghost to /ets/rsyslog.conf" do
  block do
    fh = File.open("/etc/rsyslog.conf", 'a+')
    fh.puts "#Remote logging to server Jupiter5.local.raymii.nl \n*.*     @10.5.0.69"
    fh.close
  end
  only_if { File.file?("/etc/rsyslog.conf") }
end

service "rsyslog" do
    supports :status => true, :restart => true, :reload => true
    action [ :restart ]
    only_if { File.file?("/etc/rsyslog.conf") }
end
